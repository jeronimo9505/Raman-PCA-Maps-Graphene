
Detailed Explanation of the Program: PCA Analyser Raman Spectrum

This program is designed to analyze and compare Raman spectral data from different samples organized into groups. 
The main objective is to identify significant patterns and variations using Principal Component Analysis (PCA).

---

## Functionality

### 1. Data Organization
The input data consists of Raman spectra stored as CSV files. These files are grouped into separate folders, 
each representing a comparison group. For example, folders may contain spectra from samples with different treatments, 
concentrations, or experimental conditions.

### 2. Raman Shift Range Filtering
Before performing any analysis, the program allows the user to specify a range of Raman shifts of interest (e.g., 176–300 cm⁻¹). 
This step ensures that only relevant portions of the spectrum are considered, minimizing noise and focusing on the key regions.

### 3. Global Analysis with PCA
PCA is applied globally across all spectra within the specified Raman shift range:
- **Data Preparation:** Spectra are normalized using Standard Normal Variate (SNV), ensuring consistent scaling.
- **Dimensionality Reduction:** PCA reduces the dataset to its most significant components, making it easier to interpret.
- **Global Comparison:** By combining all spectra in the PCA analysis, the program identifies patterns and similarities between groups.

### 4. Data Retagging
After the global PCA analysis, the program reassigns labels to the data points based on the groups they belong to. 
This step enables the visualization and comparison of groups in PCA plots.

### 5. Visualization
The program provides interactive visualizations:
- **Explained Variance Plot:** Shows how much variability is captured by each principal component.
- **PCA Scatter Plot (2D/3D):** Visualizes the distribution of groups in the reduced-dimensional space.
- **Representative Spectra:** Highlights key spectra that best represent each group.

## How PCA Works in This Program
1. **Mathematical Overview:**
   PCA transforms the dataset into a new coordinate system where:
   - The first axis (principal component) captures the largest variance.
   - Subsequent axes capture decreasing amounts of variance.
   - The components are orthogonal, ensuring no overlap of information.

   The formula for the PCA transformation is:
   \[
   Z = X \cdot W
   \]
   Where:
   - \( Z \): Transformed data in the PCA space.
   - \( X \): Original data matrix (e.g., Raman intensities for different shifts).
   - \( W \): Matrix of eigenvectors derived from the covariance matrix of \( X \).

2. **Steps Performed:**
   - Compute the mean-centered data matrix.
   - Calculate the covariance matrix of the data.
   - Solve for eigenvalues and eigenvectors of the covariance matrix.
   - Sort eigenvectors by eigenvalues (largest to smallest).
   - Project the data onto the eigenvectors to obtain the principal components.

3. **Why PCA?**
   PCA simplifies the dataset by reducing redundancy while preserving the most critical variations. 
   This is particularly useful for Raman data, where high dimensionality and noise can obscure meaningful trends.

## Example Workflow
1. Organize your Raman spectra into folders, with each folder representing a comparison group.
2. Run the program and specify the Raman shift range.
3. Load your data and let the program perform PCA on the combined dataset.
4. Visualize the PCA results and interpret group separations or clustering patterns.
5. Analyze representative spectra for insights into spectral features unique to each group.
